Dataset description:

Kickstarter Projects Dataset.

The given dataset is collected from Kickstarter Projects - Kaggle. The dataset contains records of projects which were initiated by an organization. The projects are categorized according to different categories and sub-categories. Other information contained is currency in which the project is funded, deadline of project submission, goal amount to be collected for its initiation, launch date and time, pledged amount which is funded by other organizations. The project could be in any state like - failed, cancelled, live, successful, suspended. Backers define how many organizations are supporting the project. Further, country of project is given. USD pledged and USD pledged real are the amounts in US dollars which are converted from goal values by different websites - KS and fixer.io respectively, similarly USD Goal real is Goal amount in USD.

Initial questions How many backers supported the projects in its deadline year? How many projects were successful to finish per year?
Description:  The above bar chart displays the count of backers for each year's project deadline. For each bar the colored sections display the state of the project. The tooltip shows the number of backers holded by each bar for a particular state of project. Value of number of backers changes according to each year.
Insight: When a project has its deadline the goals are set. To meet the goals the start-ups try to gather more backers/funders. However the existing backers represent the current pledged amount which adds up to the goal value to make a project successful. For this reason, it is important to know how many backers are involved in a project, especially when deadlines are supposed to meet.

The bar indicates that there are more backers for successful projects in these 2 years -> 2011 through 2013, capable of meeting their deadline. Less projects failed in 2011-2012; in 2012-2013, backers were the highest but few project failures and cancellations also took place. Least backers are for the year 2018-2019, where projects are still live and so far no cancellation took place but only 1 project is suspended. Maximum failures happened in 2014-2015 where backers count was also less.

Despite of having sufficient backers all along, there are a few suspensions for years 2013 through 2018.
Design considerations I considered plotting a bar graph as it is the best way to display temporal values. The choice of colors were made so that it is easy to differentiate between each state. Tooltip values shows the project state and its number of backers. This makes it easy for the user to navigate and check the value. As these values represent time as x-axis, it would be good if we would keep this plot unsorted. It is easy to view and analyze the plot on the basis of time.

Another alternative to this graph was plotting a line plot, but it was not sufficient to display the response for each state. Also, for almost accurate understanding of values a bar graph is visually relaxing.
Data filtering and transformation:  The Pledged and Goal values which are greater than zero are considered. Backers more than 100 are shown. Also, the duration of project(launched_date-deadline) is considered to be more than 50 days.
